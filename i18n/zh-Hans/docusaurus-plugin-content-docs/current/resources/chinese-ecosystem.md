# 中国生态系统

中国大陆是 OpenClaw 增长最快的市场之一，但同时也是法规最复杂的市场。从 Tencent WeChat 官方集成到国有企业使用限制，本页涵盖在中国大陆使用 OpenClaw 所需了解的一切。

:::danger 重要法规提醒
中国大陆对 AI 服务的使用有严格的法规要求，特别是国有企业。本页提供的信息仅作为参考，不构成法律建议。在部署前，请务必咨询专业法律顾问。
:::

## Tencent WeChat 集成

### 概览

Tencent 与 OpenClaw 的官方合作使得 WeChat（微信）成为 OpenClaw 支援的通信平台之一。这对中国大陆用户意义重大，因为 WeChat 是中国最普及的通信工具，拥有超过 13 亿月活跃用户。

### 集成功能

| 功能 | 支援状态 | 说明 |
|------|---------|------|
| **个人聊天** | 完整支援 | Agent 可在一对一聊天中回复 |
| **群组聊天** | 完整支援 | Agent 可参与群组对话 |
| **公众号集成** | 部分支援 | 可作为公众号的后端智慧回复 |
| **小进程交互** | 计划中 | 未来版本预计支援 |
| **WeChat Pay** | 不支援 | 涉及金融法规，不在计划内 |
| **语音消息** | 部分支援 | 需配合语音转文字技能 |

### 配置步骤

```yaml
channels:
  wechat:
    enabled: true
    app_id: "your_wechat_app_id"
    app_secret: "your_wechat_app_secret"
    token: "your_verification_token"
    encoding_aes_key: "your_aes_key"
    mode: "personal"  # 或 "official_account"
```

:::caution WeChat 配置注意事项
WeChat 集成需要通过 Tencent 的开发者认证流程，个人账号与公众号的认证要求不同。公众号集成需要企业资质。
:::

### WeChat 集成的特殊限制

1. **消息频率限制**：WeChat 对消息发送频率有严格限制
2. **内容审查**：所有经由 WeChat 的消息都会经过 Tencent 的内容审查系统
3. **数据存储**：中国法规要求相关数据必须存储在中国境内服务器
4. **账号风险**：自动化过于激进可能导致 WeChat 账号被封禁

## DeepSeek — 推荐的中国大陆 LLM

### 为什么推荐 DeepSeek？

| 优势 | 说明 |
|------|------|
| **无需翻墙** | API 在中国大陆可直接存取 |
| **价格极低** | 仅为 OpenAI 的 1/10 到 1/30 |
| **中文优化** | 针对中文场景做过特别优化 |
| **本地合规** | 符合中国大陆数据法规要求 |
| **开源模型** | 可在本地部署，数据完全可控 |

### 与 OpenClaw 的集成

```yaml
llm:
  provider: deepseek
  api_key: sk-xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
  model: deepseek-v3
  base_url: https://api.deepseek.com
```

### DeepSeek 模型比较

| 模型 | 适合场景 | 中文能力 | 价格等级 |
|------|---------|---------|---------|
| **DeepSeek-V3** | 通用对话、代码 | 优秀 | 极低 |
| **DeepSeek-R1** | 复杂推理 | 优秀 | 低 |
| **DeepSeek-Coder** | 代码专用 | 良好 | 极低 |

## 中国大陆使用限制

### 国有企业限制

:::danger 国有企业用户必读
中国大陆国有企业（State-Owned Enterprises, SOEs）在使用 AI Agent 服务时有特殊限制。以下限制可能随政策更新而变化，请定期确认最新法规。
:::

| 限制类别 | 说明 | 影响范围 |
|---------|------|---------|
| **数据存取限制** | Agent 不得存取涉及国家安全的内部数据 | 所有国有企业 |
| **模型提供者限制** | 部分情况下仅允许使用国产 LLM | 敏感产业国企 |
| **部署位置限制** | 必须部署在中国境内服务器 | 所有国有企业 |
| **审计要求** | 需定期提交 AI 使用报告 | 特定产业国企 |
| **技能白名单** | 仅允许经审核的技能 | 所有国有企业 |

### 一般企业合规要求

即使非国有企业，在中国大陆使用 OpenClaw 也需要注意：

1. **《生成式人工智能服务管理暂行办法》**
   - AI 生成的内容必须符合社会主义核心价值观
   - 需创建内容审核机制
   - 需保留使用日志至少六个月

2. **《个人信息保护法》（PIPL）**
   - 用户的对话记录属于个人信息
   - 需明确告知用户数据的收集与使用方式
   - 跨境传输数据需额外评估

3. **《数据安全法》**
   - 需对处理的数据进行安全等级分类
   - 重要数据需在中国境内存储
   - 需创建数据安全管理制度

### 合规建议

| 措施 | 说明 | 优先级 |
|------|------|--------|
| 使用国产 LLM | 选择 DeepSeek 等国产模型 | 高 |
| 境内部署 | 使用中国境内的云端服务 | 高 |
| 日志保留 | 保留至少 6 个月的完整日志 | 高 |
| 内容过滤 | 创建内容审核机制 | 高 |
| 法律咨询 | 聘请专业法律顾问 | 中 |
| 定期审计 | 每季度进行一次合规审计 | 中 |

## 推荐的中国大陆部署架构

```
┌─────────────────────────────────────────────┐
│  中国大陆云端服务器（阿里云 / 腾讯云）      │
│                                             │
│  ┌──────────────┐   ┌──────────────┐       │
│  │  OpenClaw    │   │  DeepSeek    │       │
│  │  Core        │◄──│  API         │       │
│  └──────┬───────┘   └──────────────┘       │
│         │                                   │
│  ┌──────┴───────┐   ┌──────────────┐       │
│  │  Gateway     │◄──│  WeChat      │       │
│  │  (127.0.0.1) │   │  集成        │       │
│  └──────────────┘   └──────────────┘       │
│                                             │
│  ┌──────────────┐                           │
│  │  日志系统     │  ← 保留 6 个月以上       │
│  └──────────────┘                           │
└─────────────────────────────────────────────┘
```

:::tip 部署建议
使用阿里云或腾讯云的中国大陆区域进行部署，确保满足数据在地化要求。Gateway 务必绑定到 `127.0.0.1`，透过云端供应商的负载均衡器对外服务。
:::
