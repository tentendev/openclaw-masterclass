## Top 50 必装 Skills 方法论

### 评分维度

每个技能从以下五个维度进行评分，每维度 20 分，总分 100 分：

#### 1. 实用性（20 分）

| 分数 | 标准 |
|------|------|
| 18-20 | 日常必备，几乎所有用户都需要 |
| 14-17 | 高度实用，特定场景非常有价值 |
| 10-13 | 有用但非必要 |
| 6-9 | 小众需求 |
| 0-5 | 极少使用场景 |

#### 2. 安全性（20 分）

| 分数 | 标准 |
|------|------|
| 18-20 | 最小权限，无网络/文件/shell 存取，已通过安全审查 |
| 14-17 | 有限的权限需求，开源且可审查 |
| 10-13 | 需要网络或文件存取，但域名受限 |
| 6-9 | 需要较广泛的权限 |
| 0-5 | 需要 shell 存取或万用字元权限（不推荐） |

#### 3. 稳定性（20 分）

| 分数 | 标准 |
|------|------|
| 18-20 | 稳定版本（1.0+），持续维护，极少 bug 回报 |
| 14-17 | 稳定使用，偶尔有小 bug |
| 10-13 | 可用但偶尔有问题 |
| 6-9 | 经常出现问题 |
| 0-5 | 不稳定，经常崩溃 |

#### 4. 社区评价（20 分）

| 分数 | 标准 |
|------|------|
| 18-20 | 10,000+ 安装，4.5+ 星评，社区高度推荐 |
| 14-17 | 5,000+ 安装，4.0+ 星评 |
| 10-13 | 1,000+ 安装，3.5+ 星评 |
| 6-9 | 500+ 安装 |
| 0-5 | < 500 安装 |

#### 5. 文件品质（20 分）

| 分数 | 标准 |
|------|------|
| 18-20 | 完整文件、示例、FAQ、视频教程 |
| 14-17 | 完整文件和示例 |
| 10-13 | 基本文件 |
| 6-9 | 简略文件 |
| 0-5 | 无文件或极度不完整 |

### 排名规则

1. 按总分降序排列
2. 同分时以安全性分数优先
3. 安全性仍相同时以安装数优先
4. 安全性低于 10 分的技能不列入推荐

## Top 30 Reddit Showcase 方法论

### 评选标准

每个 Showcase 从以下四个维度评分，每维度 25 分，总分 100 分：

#### 1. 社区回响（25 分）

| 指标 | 权重 |
|------|------|
| Reddit 投票数（upvotes） | 40% |
| 评论数 | 30% |
| 评论品质（实质讨论 vs 灌水） | 20% |
| 跨平台转发次数 | 10% |

#### 2. 技术创新（25 分）

| 分数 | 标准 |
|------|------|
| 20-25 | 前所未见的创新应用 |
| 15-19 | 巧妙的技术组合 |
| 10-14 | 良好的技术实现 |
| 5-9 | 标准应用 |
| 0-4 | 基础使用 |

#### 3. 可复制性（25 分）

| 分数 | 标准 |
|------|------|
| 20-25 | 附上完整步骤和配置文件，任何人可复制 |
| 15-19 | 有足够的描述，有经验者可复制 |
| 10-14 | 需要一些额外研究才能复制 |
| 5-9 | 需要特定环境或硬件 |
| 0-4 | 几乎无法复制 |

#### 4. 实际价值（25 分）

| 分数 | 标准 |
|------|------|
| 20-25 | 带来显著的时间/金钱节省或生活品质提升 |
| 15-19 | 明确的实用价值 |
| 10-14 | 有教育或启发价值 |
| 5-9 | 有趣但实用性有限 |
| 0-4 | 纯粹实验性质 |

### 来源验证

| 标记 | 说明 | 验证程度 |
|------|------|---------|
| **已验证** | 编辑团队已确认原帖存在，并核实核心事实 | 高 |
| **社区回报** | 由社区成员提供，编辑团队进行了合理性审查 | 中 |

验证流程：
1. 确认 Reddit 原帖存在且未被删除
2. 核实帖中提及的技术栈是否合理
3. 交叉比对评论中的验证信息
4. 合理性评估（成果是否在技术上可行）
