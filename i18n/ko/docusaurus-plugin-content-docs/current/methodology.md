## Top 50 必裝 Skills 方法論

### 評分維度

每個스킬從以下五個維度進行評分，每維度 20 分，總分 100 分：

#### 1. 實用性（20 分）

| 分數 | 標準 |
|------|------|
| 18-20 | 日常必備，幾乎所有사용자都需要 |
| 14-17 | 高度實用，特定場景非常有價值 |
| 10-13 | 有用但非必要 |
| 6-9 | 小眾需求 |
| 0-5 | 極少使用場景 |

#### 2. 安全性（20 分）

| 分數 | 標準 |
|------|------|
| 18-20 | 最小권한，無網路/檔案/shell 存取，已通過安全審查 |
| 14-17 | 有限的권한需求，開源且可審查 |
| 10-13 | 需要網路或檔案存取，但域名受限 |
| 6-9 | 需要較廣泛的권한 |
| 0-5 | 需要 shell 存取或萬用字元권한（不推薦） |

#### 3. 穩定性（20 分）

| 分數 | 標準 |
|------|------|
| 18-20 | 穩定版本（1.0+），持續維護，極少 bug 回報 |
| 14-17 | 穩定使用，偶爾有小 bug |
| 10-13 | 可用但偶爾有問題 |
| 6-9 | 經常出現問題 |
| 0-5 | 不穩定，經常崩潰 |

#### 4. 社群평가（20 分）

| 分數 | 標準 |
|------|------|
| 18-20 | 10,000+ 설치，4.5+ 星評，社群高度推薦 |
| 14-17 | 5,000+ 설치，4.0+ 星評 |
| 10-13 | 1,000+ 설치，3.5+ 星評 |
| 6-9 | 500+ 설치 |
| 0-5 | < 500 설치 |

#### 5. 문서品質（20 分）

| 分數 | 標準 |
|------|------|
| 18-20 | 完整문서、예시、FAQ、영상튜토리얼 |
| 14-17 | 完整문서和예시 |
| 10-13 | 基本문서 |
| 6-9 | 簡略문서 |
| 0-5 | 無문서或極度不完整 |

### 排名規則

1. 按總分降序排列
2. 同分時以安全性分數優先
3. 安全性仍相同時以설치數優先
4. 安全性低於 10 分的스킬不列入推薦

## Top 30 Reddit Showcase 方法論

### 評選標準

每個 Showcase 從以下四個維度評分，每維度 25 分，總分 100 分：

#### 1. 社群迴響（25 分）

| 指標 | 權重 |
|------|------|
| Reddit 投票數（upvotes） | 40% |
| 評論數 | 30% |
| 評論品質（實質討論 vs 灌水） | 20% |
| 跨平台轉發次數 | 10% |

#### 2. 技術創新（25 分）

| 分數 | 標準 |
|------|------|
| 20-25 | 前所未見的創新應用 |
| 15-19 | 巧妙的技術組合 |
| 10-14 | 良好的技術實作 |
| 5-9 | 標準應用 |
| 0-4 | 基礎使用 |

#### 3. 可複製性（25 分）

| 分數 | 標準 |
|------|------|
| 20-25 | 附上完整步驟和설정 파일，任何人可複製 |
| 15-19 | 有足夠的描述，有經驗者可複製 |
| 10-14 | 需要一些額外리서치才能複製 |
| 5-9 | 需要特定環境或硬體 |
| 0-4 | 幾乎無法複製 |

#### 4. 實際價值（25 分）

| 分數 | 標準 |
|------|------|
| 20-25 | 帶來顯著的時間/金錢節省或生活品質提升 |
| 15-19 | 明確的實用價值 |
| 10-14 | 有教育或啟發價值 |
| 5-9 | 有趣但實用性有限 |
| 0-4 | 純粹實驗性質 |

### 來源검증

| 標記 | 說明 | 검증程度 |
|------|------|---------|
| **已검증** | 編輯團隊已確認原帖存在，並核實核心事實 | 高 |
| **社群回報** | 由社群成員提供，編輯團隊進行了合理性審查 | 中 |

검증流程：
1. 確認 Reddit 原帖存在且未被삭제
2. 核實帖中提及的技術棧是否合理
3. 交叉比對評論中的검증資訊
4. 合理性評估（成果是否在技術上可行）
