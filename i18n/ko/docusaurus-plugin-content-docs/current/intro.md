# OpenClaw란 무엇인가? 완전 가이드

**OpenClaw**는 오픈소스 자율 AI 에이전트(Autonomous AI Agent) 플랫폼으로, 로컬 컴퓨터에서 실행되며 20개 이상의 메신저 플랫폼과 연동됩니다. 생각하고, 기억하고, 행동하는 AI 어시스턴트를 보유할 수 있으며, 모든 데이터는 사용자가 직접 관리합니다.

아시아 커뮤니티에서 OpenClaw는 **"랍스터 키우기"**라는 친근한 별명으로 불리며, 마스코트는 **Molty**라는 이름의 랍스터입니다. 이 이름은 OpenClaw의 "Claw"(집게발)에서 유래했으며, 강력하고 유연한 그래핑 능력을 상징합니다.

:::info 핵심 데이터
- GitHub Stars: **250,000+**
- ClawHub 스킬 마켓: **13,000+ 스킬**
- 지원 메신저 플랫폼: **20개 이상**
- 지원 LLM 모델: **Claude, GPT, Gemini, DeepSeek, Ollama 등**
- 창시자: **Peter Steinberger**
:::

---

## 왜 OpenClaw를 선택해야 하는가?

AI 도구가 넘쳐나는 2026년, OpenClaw가 두각을 나타내는 이유는 세 가지입니다:

1. **완전 로컬 실행**: 대화 기록, 메모리 데이터, 설정 파일 모두 로컬에 저장되며 어떤 제3자 서버에도 업로드되지 않습니다.
2. **크로스 플랫폼 통합**: 하나의 AI 에이전트로 WhatsApp, Telegram, Discord, Slack, LINE, Signal, iMessage, Matrix 등 여러 플랫폼에 동시 연결할 수 있습니다.
3. **스킬 생태계**: ClawHub 스킬 마켓을 통해 커뮤니티가 개발한 13,000개 이상의 스킬을 설치할 수 있으며, 자동 응답부터 스마트홈 제어까지 다양합니다.

## OpenClaw의 역사: Clawdbot에서 오늘까지

OpenClaw의 발전은 세 가지 주요 단계를 거쳤습니다:

### 1단계: Clawdbot (2024년 초)

Peter Steinberger가 처음 개발한 **Clawdbot**은 AI가 인스턴트 메신저를 통해 대화할 수 있게 하려는 개인 프로젝트였습니다. 당시에는 Telegram만 지원하고 단일 LLM만 호출할 수 있는 매우 기초적인 기능만 있었습니다.

### 2단계: Moltbot (2024년 중반)

커뮤니티가 빠르게 성장하면서 프로젝트 이름이 **Moltbot**(랍스터의 탈피 "molt"에서 유래)으로 변경되었고, 메모리 시스템과 다중 플랫폼 지원이 추가되었습니다. 이 단계에서 4계층 아키텍처의 기초가 마련되었습니다.

### 3단계: OpenClaw (2025년 초~현재)

공식적으로 **OpenClaw**로 이름이 바뀌면서 ClawHub 스킬 마켓, 샌드박스 실행 환경, 완전한 보안 아키텍처가 도입되었습니다. 2025년 말 200K GitHub Stars를 돌파하며 가장 빠르게 성장하는 오픈소스 AI 프로젝트 중 하나가 되었습니다.

:::tip 창시자 소식
Peter Steinberger는 2026년 2월 OpenAI에 합류했지만, OpenClaw는 오픈소스 프로젝트로서 커뮤니티 주도 개발을 이어갑니다.
:::

## 4계층 아키텍처 개요

OpenClaw는 명확한 4계층 아키텍처 설계를 채택하며, 각 계층이 고유한 역할을 담당합니다:

```
┌─────────────────────────────────────────┐
│  1계층: Gateway (게이트웨이 계층)         │
│  Port 18789 — 모든 메신저 플랫폼 메시지   │
│  통합 수신                               │
├─────────────────────────────────────────┤
│  2계층: Reasoning Layer (추론 계층)      │
│  LLM 모델 연결, 의도 인식 및 응답 생성    │
├─────────────────────────────────────────┤
│  3계층: Memory System (메모리 시스템)    │
│  WAL + Markdown 압축, 장기 기억 관리      │
├─────────────────────────────────────────┤
│  4계층: Skills / Execution Layer        │
│  (스킬 / 실행 계층) 샌드박스에서 스킬 실행 │
└─────────────────────────────────────────┘
```

### 1계층: Gateway (게이트웨이 계층)

Gateway는 OpenClaw의 진입점으로, 기본적으로 **port 18789**에서 수신 대기합니다. 각 메신저 플랫폼에서 오는 메시지를 통합 수신하여 내부 표준 형식으로 변환한 후 추론 계층에 전달합니다.

:::danger 보안 경고
Gateway의 18789 포트는 OpenClaw의 가장 큰 공격 표면입니다. 2026년 초 기준으로 **30,000개 이상의 인스턴스**가 이 포트를 공용 네트워크에 노출(`0.0.0.0` 바인딩)하여 침해당했습니다. 반드시 `127.0.0.1`에 바인딩하세요. 자세한 내용은 [보안 모범 사례](/docs/security/best-practices)를 참조하세요.
:::

### 2계층: Reasoning Layer (추론 계층)

추론 계층은 OpenClaw의 두뇌입니다. 사용자의 메시지를 설정된 LLM 모델(예: Claude Opus 4.6, GPT-5.2 Codex 등)에 전송하고, 응답을 받아 다음 행동을 결정합니다. 직접 응답하거나, 스킬을 호출하거나, 메모리를 조회할 수 있습니다.

### 3계층: Memory System (메모리 시스템)

메모리 시스템은 **WAL(Write-Ahead Log)**과 **Markdown 압축**의 하이브리드 방식을 채택합니다. 단기 기억은 WAL로 빠르게 기록하고, 장기 기억은 주기적으로 구조화된 Markdown 파일로 압축하여 효율적인 컨텍스트 관리를 실현합니다.

### 4계층: Skills / Execution Layer (스킬 / 실행 계층)

모든 스킬은 **샌드박스 환경**에서 실행되어 악성 코드가 시스템에 영향을 미치는 것을 방지합니다. 스킬은 네트워크, 파일 시스템(제한된 범위), 외부 API에 접근할 수 있지만 모두 엄격한 권한 제어를 받습니다.

:::tip 더 알아보기
아키텍처에 대해 더 자세히 알고 싶으시면 [아키텍처 개요](/docs/architecture/overview) 페이지를 방문하세요.
:::

## 보안 개요

OpenClaw의 보안 문제는 간과할 수 없습니다. 주요 보안 사건들은 다음과 같습니다:

| 사건 | 설명 |
|------|------|
| **CVE-2026-25253** | Gateway 원격 코드 실행 취약점, v3.x 이전 버전에 영향 |
| **ClawHavoc 사건** | 2,400개 이상의 악성 스킬이 ClawHub에 심어졌으며, 이후 모두 제거됨 |
| **18789 포트 노출** | 30,000개 이상의 인스턴스가 잘못된 설정으로 침해당함 |

:::warning OpenClaw 사용 전 반드시 읽으세요
보안은 선택 사항이 아닙니다. 모든 사용자는 사용을 시작하기 전에 [보안 모범 사례](/docs/security/best-practices)와 [스킬 감사 체크리스트](/docs/security/skill-audit-checklist)를 읽어야 합니다.
:::

## 누구에게 적합한가?

| 사용자 유형 | 적합한 이유 |
|-----------|---------|
| **개발자** | 스킬 직접 개발, 심층 커스터마이징, 기존 워크플로에 통합 |
| **프라이버시 중시 사용자** | 완전 로컬 실행, 데이터가 컴퓨터를 떠나지 않음 |
| **커뮤니티 관리자** | 하나의 AI로 여러 메신저 플랫폼 커뮤니티 동시 관리 |
| **자동화 매니아** | 스킬 조합으로 복잡한 자동화 워크플로 구현 |
| **기업 IT 팀** | 내부 네트워크에 배포 가능, 엔터프라이즈급 보안 설정 지원 |

간단한 챗봇만 필요하다면 상용 솔루션(ChatGPT의 기성 애플리케이션 등)이 더 적합할 수 있습니다. OpenClaw의 강점은 **심층 커스터마이징**과 **다중 플랫폼 통합**에 있습니다. 일회성 도구가 아닌, "키울 수 있는" AI 에이전트에 가깝습니다.
