# 中國生態系統

中國大陸是 OpenClaw 增長最快的市場之一，但同時也是法規最複雜的市場。從 Tencent WeChat 官方整合到國有企業使用限制，本頁涵蓋在中國大陸使用 OpenClaw 所需了解的一切。

:::danger 重要法規提醒
中國大陸對 AI 服務的使用有嚴格的法規要求，特別是國有企業。本頁提供的資訊僅作為參考，不構成法律建議。在部署前，請務必諮詢專業法律顧問。
:::

---

## 中國市場概覽

| 項目 | 狀態 |
|------|------|
| **OpenClaw 在中國的可用性** | 可用（開源軟體，可自行部署） |
| **ClawHub 存取** | 需要翻牆或使用國內映像 |
| **官方 WeChat 整合** | 已完成（Tencent 官方合作） |
| **推薦 LLM 提供者** | DeepSeek（無需翻牆） |
| **國有企業使用** | 有限制，需合規審查 |
| **個人使用** | 無明確限制 |

## Tencent WeChat 整合

### 概覽

Tencent 與 OpenClaw 的官方合作使得 WeChat（微信）成為 OpenClaw 支援的通訊平台之一。這對中國大陸使用者意義重大，因為 WeChat 是中國最普及的通訊工具，擁有超過 13 億月活躍用戶。

### 整合功能

| 功能 | 支援狀態 | 說明 |
|------|---------|------|
| **個人聊天** | 完整支援 | Agent 可在一對一聊天中回覆 |
| **群組聊天** | 完整支援 | Agent 可參與群組對話 |
| **公眾號整合** | 部分支援 | 可作為公眾號的後端智慧回覆 |
| **小程序互動** | 計畫中 | 未來版本預計支援 |
| **WeChat Pay** | 不支援 | 涉及金融法規，不在計畫內 |
| **語音訊息** | 部分支援 | 需配合語音轉文字技能 |

### 設定步驟

```yaml
channels:
  wechat:
    enabled: true
    app_id: "your_wechat_app_id"
    app_secret: "your_wechat_app_secret"
    token: "your_verification_token"
    encoding_aes_key: "your_aes_key"
    mode: "personal"  # 或 "official_account"
```

:::caution WeChat 設定注意事項
WeChat 整合需要通過 Tencent 的開發者認證流程，個人帳號與公眾號的認證要求不同。公眾號整合需要企業資質。
:::

### WeChat 整合的特殊限制

1. **訊息頻率限制**：WeChat 對訊息發送頻率有嚴格限制
2. **內容審查**：所有經由 WeChat 的訊息都會經過 Tencent 的內容審查系統
3. **資料儲存**：中國法規要求相關資料必須儲存在中國境內伺服器
4. **帳號風險**：自動化過於激進可能導致 WeChat 帳號被封禁

## DeepSeek — 推薦的中國大陸 LLM

### 為什麼推薦 DeepSeek？

| 優勢 | 說明 |
|------|------|
| **無需翻牆** | API 在中國大陸可直接存取 |
| **價格極低** | 僅為 OpenAI 的 1/10 到 1/30 |
| **中文優化** | 針對中文場景做過特別優化 |
| **本地合規** | 符合中國大陸資料法規要求 |
| **開源模型** | 可在本地部署，資料完全可控 |

### 與 OpenClaw 的整合

```yaml
llm:
  provider: deepseek
  api_key: sk-xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
  model: deepseek-v3
  base_url: https://api.deepseek.com
```

### DeepSeek 模型比較

| 模型 | 適合場景 | 中文能力 | 價格等級 |
|------|---------|---------|---------|
| **DeepSeek-V3** | 通用對話、程式碼 | 優秀 | 極低 |
| **DeepSeek-R1** | 複雜推理 | 優秀 | 低 |
| **DeepSeek-Coder** | 程式碼專用 | 良好 | 極低 |

## 中國大陸使用限制

### 國有企業限制

:::danger 國有企業使用者必讀
中國大陸國有企業（State-Owned Enterprises, SOEs）在使用 AI Agent 服務時有特殊限制。以下限制可能隨政策更新而變化，請定期確認最新法規。
:::

| 限制類別 | 說明 | 影響範圍 |
|---------|------|---------|
| **資料存取限制** | Agent 不得存取涉及國家安全的內部資料 | 所有國有企業 |
| **模型提供者限制** | 部分情況下僅允許使用國產 LLM | 敏感產業國企 |
| **部署位置限制** | 必須部署在中國境內伺服器 | 所有國有企業 |
| **審計要求** | 需定期提交 AI 使用報告 | 特定產業國企 |
| **技能白名單** | 僅允許經審核的技能 | 所有國有企業 |

### 一般企業合規要求

即使非國有企業，在中國大陸使用 OpenClaw 也需要注意：

1. **《生成式人工智能服務管理暫行辦法》**
   - AI 生成的內容必須符合社會主義核心價值觀
   - 需建立內容審核機制
   - 需保留使用日誌至少六個月

2. **《個人信息保護法》（PIPL）**
   - 使用者的對話記錄屬於個人資訊
   - 需明確告知使用者資料的收集與使用方式
   - 跨境傳輸資料需額外評估

3. **《數據安全法》**
   - 需對處理的資料進行安全等級分類
   - 重要資料需在中國境內儲存
   - 需建立資料安全管理制度

### 合規建議

| 措施 | 說明 | 優先級 |
|------|------|--------|
| 使用國產 LLM | 選擇 DeepSeek 等國產模型 | 高 |
| 境內部署 | 使用中國境內的雲端服務 | 高 |
| 日誌保留 | 保留至少 6 個月的完整日誌 | 高 |
| 內容過濾 | 建立內容審核機制 | 高 |
| 法律諮詢 | 聘請專業法律顧問 | 中 |
| 定期審計 | 每季度進行一次合規審計 | 中 |

## 推薦的中國大陸部署架構

```
┌─────────────────────────────────────────────┐
│  中國大陸雲端伺服器（阿里雲 / 騰訊雲）      │
│                                             │
│  ┌──────────────┐   ┌──────────────┐       │
│  │  OpenClaw    │   │  DeepSeek    │       │
│  │  Core        │◄──│  API         │       │
│  └──────┬───────┘   └──────────────┘       │
│         │                                   │
│  ┌──────┴───────┐   ┌──────────────┐       │
│  │  Gateway     │◄──│  WeChat      │       │
│  │  (127.0.0.1) │   │  整合        │       │
│  └──────────────┘   └──────────────┘       │
│                                             │
│  ┌──────────────┐                           │
│  │  日誌系統     │  ← 保留 6 個月以上       │
│  └──────────────┘                           │
└─────────────────────────────────────────────┘
```

:::tip 部署建議
使用阿里雲或騰訊雲的中國大陸區域進行部署，確保滿足資料在地化要求。Gateway 務必綁定到 `127.0.0.1`，透過雲端供應商的負載均衡器對外服務。
:::
